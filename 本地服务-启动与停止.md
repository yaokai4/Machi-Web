# Machi 本地服务 — 启动 / 停止 / 排查 指南

本地需要**两个服务**同时运行:

| 服务 | 地址 | 启动目录 | 说明 |
|---|---|---|---|
| Python 后端 | http://127.0.0.1:8787 | 仓库根 `kaizi/` | API + SQLite(`web/kaix.db`) |
| Next.js 前端 | http://localhost:3000 | `web/app/` | 网页,`/api` 反向代理到后端 |

> 仓库根固定为 `/Users/yaokai/Desktop/IT/IOS/kaizi`(下文用 `$ROOT` 代指)。

---

## ⚠️ 最重要的一条(上次登录全失败的原因)

后端默认读 **`web/kaix.db`**(你的真库,130 个账号)。
**绝对不要**把 `KAIX_DB_PATH` 指到 `/tmp/...` 之类的临时库去启动后端 —— 上次就是因为后端被启动到 `/tmp/kaizi-japan-crawler-browser.db`(只有 6 个 demo 账号、没有 yaokai),导致"任何账号都登不进"。

同理:本地开发**不要**设 `KAIX_PASSWORD_PEPPER`。真库的密码是用默认 pepper 加密的;一旦设了别的值,**所有人都登不进**(pepper 改了 = 旧密码全失效)。

## 1. 启动

### 1A. 后端(二选一)

**方式一:自己的终端(推荐,日志直接看,Ctrl-C 停)**
```bash
cd /Users/yaokai/Desktop/IT/IOS/kaizi
python3 -u web/server.py
```
看到 `Machi backend starting on http://127.0.0.1:8787` 即成功。

**方式二:后台常驻(关掉终端也不停)**
```bash
cd /Users/yaokai/Desktop/IT/IOS/kaizi
nohup env -u KAIX_DB_PATH python3 -u web/server.py > /tmp/machi_backend.log 2>&1 &
disown
```
> `env -u KAIX_DB_PATH` 是保险:确保即使当前 shell 残留了那个临时库变量,也会被清掉、用回真库。

### 1B. 前端(二选一)

**方式一:开发模式(推荐,启动快、自动热更新、能即时看到新页面)**
```bash
cd /Users/yaokai/Desktop/IT/IOS/kaizi/web/app
npm run dev
```

**方式二:生产模式(更接近线上,需先 build)**
```bash
cd /Users/yaokai/Desktop/IT/IOS/kaizi/web/app
npm run build && npm run start
```

后台常驻版(开发模式):
```bash
cd /Users/yaokai/Desktop/IT/IOS/kaizi/web/app
nohup npm run dev > /tmp/machi_frontend.log 2>&1 &
disown
```

启动后访问 **http://localhost:3000**。

> 首次没装依赖时先跑一次:`cd /Users/yaokai/Desktop/IT/IOS/kaizi/web/app && npm install`

## 2. 查看状态

```bash
# 端口是否在监听(有 PID = 在跑)
lsof -ti:8787   # 后端
lsof -ti:3000   # 前端

# 后端健康检查(返回 {"ok":true...} 即正常)
curl -s http://127.0.0.1:8787/healthz

# 实测登录(返回 token = 正常;返回 invalid_credentials = 多半连错库)
curl -s -X POST http://127.0.0.1:8787/api/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"handle":"yaokai","password":"123456"}'

# 确认后端连的是真库:应能查到 yaokai
sqlite3 'file:/Users/yaokai/Desktop/IT/IOS/kaizi/web/kaix.db?mode=ro' \
  "SELECT handle FROM users WHERE handle='yaokai';"
```

## 3. 看日志(后台模式)
```bash
tail -f /tmp/machi_backend.log     # 后端
tail -f /tmp/machi_frontend.log    # 前端
```
前台模式直接看终端输出即可。

## 4. 停止

**前台模式**:在对应终端按 `Ctrl-C`。

**后台模式 / 不确定 PID 时,按端口杀(最稳)**:
```bash
lsof -ti:8787 | xargs kill        # 停后端
lsof -ti:3000 | xargs kill        # 停前端
# 杀不掉再加 -9:
lsof -ti:8787 | xargs kill -9
lsof -ti:3000 | xargs kill -9
```

一键全停:
```bash
lsof -ti:8787 -ti:3000 | xargs kill 2>/dev/null; echo "已停止本地前后端"
```

## 5. 重启
```bash
# 后端
lsof -ti:8787 | xargs kill -9 2>/dev/null
cd /Users/yaokai/Desktop/IT/IOS/kaizi
nohup env -u KAIX_DB_PATH python3 -u web/server.py > /tmp/machi_backend.log 2>&1 & disown

# 前端
lsof -ti:3000 | xargs kill -9 2>/dev/null
cd /Users/yaokai/Desktop/IT/IOS/kaizi/web/app
nohup npm run dev > /tmp/machi_frontend.log 2>&1 & disown
```

## 6. 可选:存成脚本

把下面三个存到 `web/` 下,`chmod +x web/*.sh` 后即可 `./web/dev-start.sh` 等使用。

`web/dev-start.sh`
```bash
#!/usr/bin/env bash
set -e
ROOT="/Users/yaokai/Desktop/IT/IOS/kaizi"
lsof -ti:8787 | xargs kill -9 2>/dev/null || true
lsof -ti:3000 | xargs kill -9 2>/dev/null || true
( cd "$ROOT" && nohup env -u KAIX_DB_PATH python3 -u web/server.py > /tmp/machi_backend.log 2>&1 & disown )
( cd "$ROOT/web/app" && nohup npm run dev > /tmp/machi_frontend.log 2>&1 & disown )
echo "已启动:后端 http://127.0.0.1:8787 · 前端 http://localhost:3000"
echo "日志:tail -f /tmp/machi_backend.log /tmp/machi_frontend.log"
```

`web/dev-stop.sh`
```bash
#!/usr/bin/env bash
lsof -ti:8787 | xargs kill 2>/dev/null || true
lsof -ti:3000 | xargs kill 2>/dev/null || true
echo "已停止本地前后端"
```

`web/dev-status.sh`
```bash
#!/usr/bin/env bash
echo "后端 :8787 -> $(lsof -ti:8787 || echo 未运行)"
echo "前端 :3000 -> $(lsof -ti:3000 || echo 未运行)"
curl -s -m3 http://127.0.0.1:8787/healthz || echo "后端无响应"
```

## 7. 常见问题排查

| 现象 | 原因 / 解决 |
|---|---|
| **任何账号都登不进 / invalid_credentials** | 后端连错了库(`KAIX_DB_PATH` 指到临时库),或设了不同的 `KAIX_PASSWORD_PEPPER`。→ 不带 `KAIX_DB_PATH`、不带 `KAIX_PASSWORD_PEPPER` 重启后端。用上面"确认连真库"命令核对。 |
| **网页打不开 / `Application error` / 裸 HTML** | 前端没起,或代理不到后端。→ 确认 `:3000` 在跑、`:8787` 也在跑(前端 `/api` 会转发到 8787)。 |
| **`address already in use` / 端口被占** | 旧进程没退干净。→ `lsof -ti:8787 | xargs kill -9` 后再启。 |
| **改了代码网页没变** | 生产模式(`npm run start`)不会热更新。→ 用 `npm run dev`,或重新 `npm run build && npm run start`。 |
| **后端起不来报缺模块** | 确认在仓库根用 `python3 web/server.py` 启动;`web/seed_content_library.py`、`web/services/` 要和 `server.py` 在一起。 |
| **前端报缺依赖** | `cd web/app && npm install` 后再启。 |

## 8. 当前已知运行参数(本机默认)
- 后端端口:`KAIX_PORT`(默认 `8787`)
- 数据库:`KAIX_DB_PATH`(**默认 = `web/kaix.db`,本地保持默认即可**)
- 前端 → 后端代理:`web/app/next.config.mjs` 里的 `NEXT_PUBLIC_API_BASE`(默认 `http://127.0.0.1:8787`)
- 本地无需配置邮件 / 支付 / GeoIP / pepper,默认值即可跑通登录与浏览。
